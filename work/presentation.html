Signal Intelligence

What the Dashboard Tracks

Four early-warning signals that were scattered across 40 dashboards — now in one place.

📉

Actual Overall Churn

Monthly churn % at both logo level and account level, tracked together. Every number carries its definition version — logo gross, revenue gross, revenue net, and finance definitions are all available but clearly labelled so no one confuses them.

🔄

Subscription Changes

Upgrades and downgrades at the logo level. A Paramount+ downgrade from Premium to Standard is a churn signal — the Finance definition counts it; others don't. We surface both interpretations and flag which definition produced each number.

📅

Contract End Dates

Exact expiry dates for every logo subscription with configurable alert windows (default: 30 days out). Grace-period customers — those churning within 14 calendar days of contract end — are flagged separately so the ops team can act before the window closes.

🔔

Renewals Needed

Propensity-scored renewal pipeline. At-risk MRR = expiring contract value × predicted churn rate from the propensity model. Sarah sees which logos to prioritise this week — not a list of 200, but a ranked queue of the ones most likely to lapse.

The Three Rules That Lock It In
1

Version Everything

Every result carries definition_version: churn_v1. No number leaves the engine without provenance. When the definition changes, the version bumps and historical results stay queryable.

2

No Vague Thresholds

"Recent" = 14 calendar days. "Significant downgrade" = ARR delta > 10%. "Material churn" = > 0.5% of active base. Every cut-off is explicit, auditable, and stored in the definition file — not buried in a BI tool someone built three years ago.

3

Edge Cases Before Happy Path

Grace periods, mid-period plan changes, and partial cancellations are defined in writing before any code runs. The reconciliation table in the next room proves we tested the hard cases — not just the ones everyone agrees on.
